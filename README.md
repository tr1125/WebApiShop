הוראות התקנת הDB:

הנה המדריך המפורט לשחזור מסד הנתונים (Restore) במחשב היעד, שלב אחר שלב, כדי לוודא שכל התוכן עובר בצורה מושלמת:

שלב 1: הכנת הקובץ
העתק את קובץ ה-.bak שיצרת במחשב המקור אל מחשב היעד.

טיפ חשוב: מומלץ להניח את הקובץ בתיקיית הגיבויים כברירת מחדל של SQL Server (בדרך כלל בנתיב C:\Program Files\Microsoft SQL Server\MSSQL... \Backup), זה מונע בעיות של הרשאות גישה לקובץ.

שלב 2: פתיחת אשף השחזור ב-SSMS
פתח את SQL Server Management Studio במחשב החדש והתחבר לשרת.

לחץ קליק ימני על תיקיית Databases בסייר האובייקטים (Object Explorer).

בחר באפשרות Restore Database....

שלב 3: בחירת המקור (Source)
בחלון שנפתח, תחת הסעיף Source, סמן את האפשרות Device.

לחץ על כפתור שלוש הנקודות (...) שנמצא בצד ימין.

בחלונית הקטנה שנפתחה, ודא שסוג הקובץ הוא "File" ולחץ על Add.

נוט בתיקיות עד שתמצא את קובץ ה-.bak שהעברת, בחר אותו ולחץ על OK.

לחץ שוב על OK בחלונית בחירת המכשיר. כעת המערכת תסרוק את הגיבוי ותציג אותו ברשימה למטה.

שלב 4: הגדרות חשובות (Files & Options)
לפני שאתה לוחץ על אישור סופי, כדאי לעבור על שתי לשוניות בצד שמאל:

לשונית Files: ודא שסומן ה-V בתיבת Relocate all files to folder. זה חשוב מאוד אם נתיבי התיקיות במחשב החדש שונים מהמחשב הישן (למשל שם משתמש שונה ב-Windows).

לשונית Options:

אם כבר קיים DB בשם הזה ואתה רוצה לדרוס אותו, סמן Overwrite the existing database (WITH REPLACE).

וודא שסומן Close existing connections to destination database (כדי שהשחזור לא ייכשל בגלל שמישהו "תופס" את מסד הנתונים).

שלב 5: סיום
לחץ על OK בתחתית החלון.

המתן לסיום התהליך. בסיום תופיע הודעה: Database 'YourDBName' restored successfully.

מה השלב הבא?
לאחר השחזור, כדאי לרענן (Refresh) את תיקיית ה-Databases ב-SSMS. כעת תוכל לפתוח את הטבלאות ולראות שכל המידע שם.
